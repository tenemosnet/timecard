# 勤怠管理 管理者ページ — 設計仕様書

**作成日:** 2026年4月5日
**前提:** タイムカード設計仕様書（既存GASシステム）の拡張

---

## 1. システム概要

既存のタイムカードシステム（GAS + Googleスプレッドシート）に、管理者向けのPDF管理機能を追加する。
管理者ページはエックスサーバー上にPHP + MySQLで構築し、Googleドライブ上のPDFを参照・ダウンロードできるようにする。

### 1.1 追加する機能

| 機能 | 概要 |
|------|------|
| PDF自動生成 | 管理者ページからワンクリックで、指定スタッフ・指定月のPDFを生成 |
| PDF保管 | 生成したPDFをGoogleドライブの専用フォルダに自動保存 |
| PDF一覧・ダウンロード | スタッフ名・年月を指定してPDFを検索・ダウンロード |
| ログイン認証 | 総務担当者1名のみアクセス可能 |

### 1.2 変更しないもの

- タブレット打刻画面（Index.html）→ そのまま
- 打刻ログの記録処理（recordClock）→ そのまま
- スプレッドシートの構成・関数 → そのまま
- スタッフの日常操作 → 一切変更なし

## 3. GAS側の追加機能

既存のCode.gsに以下の関数を追加する。

### 3.1 追加関数一覧

| 関数名 | 呼び出し元 | 説明 |
|--------|-----------|------|
| `doPost(e)` | 管理者ページ（HTTP POST） | APIエンドポイント（PDF生成・一覧取得） |
| `generatePDF(staffName, year, month)` | doPost内部 | スタッフシートをPDF化してドライブに保存 |
| `listPDFs(staffName, year)` | doPost内部 | ドライブ内のPDF一覧を返す |
| `getPDFDownloadUrl(fileId)` | doPost内部 | PDFのダウンロード用URLを返す |

### 3.2 doPost の仕様

管理者ページからのHTTP POSTリクエストを受け付けるAPIエンドポイント。
簡易認証として、リクエストにAPIキー（共有シークレット）を含める。

**リクエスト形式:**
```json
{
  "action": "generatePDF" | "listPDFs" | "getPDFUrl",
  "apiKey": "（事前に決めた秘密の文字列）",
  "staffName": "山田 太郎",
  "year": 2026,
  "month": 4
}
```

**レスポンス形式:**
```json
{
  "success": true,
  "data": { ... }
}
```

### 3.3 generatePDF の仕様

**処理フロー:**
1. 指定されたスタッフ名のシートを取得
2. シートのD1（年）とF1（月）が指定の年月と一致するか確認
   - 一致しない場合、一時的に年月を変更 → PDF生成 → 元に戻す
   - ※ 打刻ログの生データは年月に関係なく保持されているため、過去月のPDFも生成可能
3. スプレッドシートのエクスポート機能でPDFを生成
4. Googleドライブの指定フォルダに保存

**PDFファイル名:** `山田太郎_2026年04月_勤怠表.pdf`

**Googleドライブのフォルダ構成:**
```
勤怠管理PDF/
  ├── 2026/
  │   ├── 04/
  │   │   ├── 山田太郎_2026年04月_勤怠表.pdf
  │   │   ├── 佐藤花子_2026年04月_勤怠表.pdf
  │   │   └── 田中一郎_2026年04月_勤怠表.pdf
  │   ├── 05/
  │   │   └── ...
  │   └── ...
  └── ...
```

**GASでのPDF生成方法（技術メモ）:**
```javascript
// スプレッドシートの特定シートをPDFとしてエクスポート
function generatePDF(staffName, year, month) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName(staffName);
  const ssId = ss.getId();
  const sheetId = sheet.getSheetId();

  // PDF出力のオプション
  const url = 'https://docs.google.com/spreadsheets/d/' + ssId +
    '/export?format=pdf' +
    '&gid=' + sheetId +
    '&size=A4' +
    '&portrait=true' +
    '&fitw=true' +           // 幅に合わせる
    '&gridlines=false' +     // グリッド線なし
    '&printtitle=false' +
    '&sheetnames=false' +
    '&fzr=true' +            // 固定行を含める
    '&range=A1:H41';         // 印刷範囲（明細＋集計まで）

  const options = {
    headers: { Authorization: 'Bearer ' + ScriptApp.getOAuthToken() }
  };

  const response = UrlFetchApp.fetch(url, options);
  const pdfBlob = response.getBlob().setName(staffName + '_' + year + '年' + month + '月_勤怠表.pdf');

  // フォルダに保存
  const folder = getOrCreateFolder_(year, month);
  const existing = folder.getFilesByName(pdfBlob.getName());
  if (existing.hasNext()) existing.next().setTrashed(true); // 既存を削除
  const file = folder.createFile(pdfBlob);

  return {
    fileId: file.getId(),
    fileName: file.getName(),
    url: file.getUrl()
  };
}
```

### 3.5 APIキー認証

簡易的なAPIキー認証を導入する。

**設定方法:**
- GASのスクリプトプロパティに `API_KEY` を設定
- 管理者ページからのリクエストに同じキーを含める
- doPost内でキーを照合、不一致なら拒否

```javascript
const API_KEY = PropertiesService.getScriptProperties().getProperty('API_KEY');

function doPost(e) {
  const params = JSON.parse(e.postData.contents);
  if (params.apiKey !== API_KEY) {
    return ContentService.createTextOutput(
      JSON.stringify({ success: false, error: '認証エラー' })
    ).setMimeType(ContentService.MimeType.JSON);
  }
  // ... 処理
}
```

### 4.5 PDFダウンロード処理（download.php）

管理者ページからGoogleドライブのPDFをダウンロードする流れ:

1. 管理者が[DL]をクリック
2. download.phpがGAS APIを呼び出し、PDFのダウンロードURLを取得
3. GASがGoogleドライブからPDFのバイナリを取得して返す
4. download.phpがブラウザにPDFを返す（Content-Disposition: attachment）

**※ 直接Googleドライブから取得する代替案:**
GASにPDFのバイナリをBase64で返す関数を追加し、PHPでデコードしてダウンロードさせる。
ファイルサイズが小さい（勤怠表は数十KB）ため、この方式でも問題ない。

```javascript
// GAS側: PDFバイナリをBase64で返す
function getPDFContent(fileId) {
  const file = DriveApp.getFileById(fileId);
  const blob = file.getBlob();
  return {
    fileName: file.getName(),
    mimeType: blob.getContentType(),
    base64: Utilities.base64Encode(blob.getBytes())
  };
}
```

```php
// PHP側: Base64をデコードしてダウンロード
$response = callGasApi('getPDFContent', ['fileId' => $fileId]);
$pdfData = base64_decode($response['data']['base64']);

header('Content-Type: application/pdf');
header('Content-Disposition: attachment; filename="' . $response['data']['fileName'] . '"');
header('Content-Length: ' . strlen($pdfData));
echo $pdfData;
```

### 6.2 PHP側のAPI呼び出し関数

```php
// api.php
function callGasApi(string $action, array $params = []): array {
    $url = GAS_API_URL;  // config.phpで定義
    $payload = array_merge([
        'action' => $action,
        'apiKey' => GAS_API_KEY,  // config.phpで定義
    ], $params);

    $ch = curl_init($url);
    curl_setopt_array($ch, [
        CURLOPT_POST => true,
        CURLOPT_POSTFIELDS => json_encode($payload),
        CURLOPT_HTTPHEADER => ['Content-Type: application/json'],
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_FOLLOWLOCATION => true,  // GASはリダイレクトする
        CURLOPT_TIMEOUT => 120,          // PDF生成は時間がかかる場合がある
    ]);

    $response = curl_exec($ch);
    $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    curl_close($ch);

    if ($httpCode !== 200) {
        throw new Exception("GAS API error: HTTP {$httpCode}");
    }

    return json_decode($response, true);
}
```

## 7. セキュリティ設計

### 7.1 通信の安全性

| 経路 | 対策 |
|------|------|
| 総務PC ↔ エックスサーバー | HTTPS必須（.htaccessで強制） |
| エックスサーバー ↔ GAS | HTTPS（GASのURLは常にHTTPS） |
| GAS ↔ Googleドライブ | Google内部通信（暗号化済み） |

### 7.2 認証・認可

| 対象 | 方式 |
|------|------|
| 管理者ページへのアクセス | PHPセッション + ID/パスワード |
| GAS APIへのアクセス | APIキー（共有シークレット） |
| Googleドライブ | GASのサービスアカウント経由（直接公開しない） |

### 7.3 .htaccess 設定

```apache
# HTTPS強制
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [R=301,L]

# ディレクトリリスティング禁止
Options -Indexes

# config.phpへの直接アクセス禁止
<Files "config.php">
  Order Allow,Deny
  Deny from all
</Files>

<Files "auth.php">
  Order Allow,Deny
  Deny from all
</Files>

<Files "api.php">
  Order Allow,Deny
  Deny from all
</Files>
```

- スタッフの追加・削除を管理者ページから操作
- 打刻データの閲覧・修正機能（スプレッドシートを開かずに管理者ページ上で完結）

## 11. 技術メモ

### 11.1 GASのPDF生成制限

- 1回のリクエストで6分のタイムアウトあり（1名分のPDFなら数秒で完了）
- 全スタッフ分を一括生成する場合は、1名ずつ順番にリクエストを送る（PHP側でループ）
- GASの1日のURL Fetchの上限: 20,000回（無料アカウント）→ 十分余裕あり

### 11.2 エックスサーバーの注意点

- PHPのcurl拡張が有効であること（通常はデフォルトで有効）
- PHPのmax_execution_timeを確認（PDF生成で時間がかかる場合）
- SSL証明書は無料のLet's Encryptが利用可能（エックスサーバーの管理画面から設定）

### 11.3 Googleドライブの容量

- 無料のGoogleアカウント: 15GB
- 勤怠表PDF: 1枚あたり約50〜100KB
- 5名×12ヶ月×100KB = 年間6MB → 容量の心配は不要

### 11.4 APIキーの安全な管理

- GAS側: スクリプトプロパティに保存（コードにハードコーディングしない）
- PHP側: config.phpに保存（.htaccessでWebからの直接アクセスを禁止）
- APIキーは32文字以上のランダム文字列を推奨
- 生成方法: `openssl rand -base64 32` または PHP `bin2hex(random_bytes(32))`
