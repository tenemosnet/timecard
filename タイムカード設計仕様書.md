# タイムカード入退勤管理システム — 設計仕様書

**作成日:** 2026年4月5日
**ステータス:** VS Code / Claude Code での実装引き継ぎ用

---

## 1. システム概要

小規模事業所（5名以下）向けの入退勤打刻・勤務時間集計システム。
専用タブレット端末にブラウザで表示した打刻画面から、スタッフが自分の名前をタップして入室・退室を記録する。データはGoogleスプレッドシートに保存され、月末に総務が確認・修正・集計を行う。

---

## 2. システム構成

### 2.1 全体アーキテクチャ

```
[タブレット端末]  --HTTP POST-->  [Google Apps Script]  --書き込み-->  [Googleスプレッドシート]
  (ブラウザ)                       (Webアプリとして公開)                  (データ保存・集計)
                                                                              ↑
                                                                     [総務PC] ブラウザで直接アクセス
```

### 2.2 構成ファイル

| ファイル | 役割 | 設置場所 |
|---------|------|---------|
| Code.gs | サーバー側ロジック（打刻受付、初期設定、月次管理） | Google Apps Script エディタ |
| Index.html | タブレット用打刻画面（フロントエンド） | Google Apps Script エディタ内のHTMLファイル |
| Googleスプレッドシート | データ保存・自動集計 | Googleドライブ |

### 2.3 デプロイ方法

1. Googleスプレッドシートを新規作成
2. スプレッドシートのURLからIDをコピー（`https://docs.google.com/spreadsheets/d/【このIDをコピー】/edit`）
3. 「拡張機能 → Apps Script」からエディタを開く
4. Code.gs にサーバーコードを貼り付け、**`SPREADSHEET_ID` に手順2のIDを設定**
5. 「ファイル → 新規 → HTML」で `Index` という名前のHTMLファイルを作成し、フロントエンドコードを貼り付け
6. 保存 → スプレッドシートをリロード → メニュー「勤怠管理 → 初期設定」を実行
7. 「デプロイ → 新しいデプロイ → ウェブアプリ」で公開
   - 実行するユーザー: 自分
   - アクセスできるユーザー: 全員
8. 発行されたURLをタブレット端末のブラウザで開く

## 3. スプレッドシート構成

### 3.1 シート一覧

| シート名 | 役割 | 操作者 |
|---------|------|-------|
| 打刻ログ | 全スタッフの生打刻データ（1行＝1打刻イベント） | 触らない（アプリが自動追記） |
| 山田 太郎 | 山田さんの月次勤務表 | 総務が確認・修正 |
| 佐藤 花子 | 佐藤さんの月次勤務表 | 同上 |
| 田中 一郎 | 田中さんの月次勤務表 | 同上 |

※ スタッフ1人＝1シート。シート名＝スタッフ名。

### 3.2 打刻ログシートの列構成

| 列 | 内容 | 書式 |
|----|------|------|
| A: 記録日時 | 打刻された正確な日時 | yyyy/MM/dd HH:mm |
| B: 氏名 | スタッフ名 | テキスト |
| C: 種別 | 「入室」または「退室」 | テキスト |
| D: 日付 | 記録日（時刻なし、集計用キー） | yyyy/MM/dd |

### 3.3 スタッフ別シートの構成

#### ヘッダー部（行1）
- A1: スタッフ名（太字・13pt）
- C1: 「年：」
- D1: 年（数値、例: 2026）
- E1: 「月：」
- F1: 月（数値、例: 4）

※ D1とF1の値を変更するだけで、表示月を切り替え可能。

#### ヘッダー（行3）
| A | B | C | D | E | F | G | H |
|---|---|---|---|---|---|---|---|
| 日付 | 曜日 | 入室 | 退室 | 実働(h) | 通常(h) | 残業(h) | 備考 |

#### 日別データ（行4〜34、最大31日分）

| 列 | 内容 | 数式ロジック |
|----|------|------------|
| A: 日付 | 月の日数に応じて自動生成 | `=IF(日番号<=DAY(EOMONTH(DATE(D1,F1,1),0)),DATE(D1,F1,日番号),"")` |
| B: 曜日 | 日本語曜日 | `=IF(A行="","",TEXT(A行,"aaa"))` ※スプレッドシートの地域設定が「日本」であること |
| C: 入室 | 打刻ログから自動取得 | `=IFERROR(INDEX(FILTER(打刻ログ!A:A, 打刻ログ!D:D=A行, 打刻ログ!B:B=$A$1, 打刻ログ!C:C="入室"),1),"")` |
| D: 退室 | 打刻ログから自動取得 | 同上（種別＝"退室"） |
| E: 実働(h) | 昼休み控除後の実働時間 | `=IF(入退室あり, LET(raw,(退室-入室)*24, lunch,IF(raw>=8,1,IF(raw>=6,0.75,0)), MAX(raw-lunch,0)), "")` |
| F: 通常(h) | 8時間を上限とした通常勤務 | `=IF(E行="","",MIN(E行,8))` |
| G: 残業(h) | 8時間超過分 | `=IF(E行="","",MAX(E行-8,0))` |
| H: 備考 | 手動入力欄 | 空白（総務が有給・打刻漏れ修正等を記入） |

#### 月次集計セクション（行36〜41）

| 行 | 項目 | 数式 |
|----|------|------|
| 36 | 【月次集計】（セクションタイトル） | — |
| 37 | 出勤日数 | `=COUNTA(FILTER(C4:C34,C4:C34<>""))` |
| 38 | 通常勤務合計 | `=SUM(F4:F34)` |
| 39 | 残業合計 | `=SUM(G4:G34)` |
| 40 | 昼休み控除合計 | `=SUMPRODUCT(...)` |
| 41 | 総勤務時間 | `=B38+B39`（通常＋残業） |

#### 書式設定
- 行3（ヘッダー）: 太字、背景色 #e2e8f0、行固定
- 土曜日の行: 文字色 #2563eb、背景 #eff6ff（条件付き書式）
- 日曜日の行: 文字色 #dc2626、背景 #fef2f2（条件付き書式）
- 行41（総勤務時間）: 太字、背景 #dbeafe
- 時刻列: H:mm 形式
- 実働・通常・残業: 0.00 形式

## 4. サーバー機能（Code.gs）

### 4.1 関数一覧

| 関数名 | 種類 | 説明 |
|--------|------|------|
| `onOpen()` | トリガー | スプレッドシートのメニューに「勤怠管理」を追加 |
| `doGet()` | Webアプリ | Index.htmlを返す（打刻画面の表示） |
| `initialize()` | メニュー | 初期設定：打刻ログシート＋スタッフ別シートを自動生成 |
| `recordClock(staffName, type)` | API | 打刻を受け付けて打刻ログに1行追加 |
| `getStaffWithStatus()` | API | スタッフ一覧と本日の打刻ステータスを返す |
| `promptNewMonth()` | メニュー | 全スタッフシートの年月を更新 |
| `showStaffHelp()` | メニュー | スタッフ名変更・追加の手順をダイアログ表示 |

### 4.2 recordClock の仕様

**入力:** staffName (string), type (string: "入室" or "退室")

**処理:**
1. 打刻ログシートの存在を確認
2. 重複チェック：同じ日・同じスタッフ・同じ種別の打刻が既にあればエラー
3. 打刻ログに [現在日時, 氏名, 種別, 今日の日付] を追記
4. セルの書式を設定

**戻り値:** `{ success: true, message: "〇〇さんの入室を記録しました", time: "09:02" }`

**エラー時:** throw new Error("〇〇さんは本日すでに入室打刻済みです")

### 4.3 getStaffWithStatus の仕様

**処理:**
1. スプレッドシートの全シート名を取得（「打刻ログ」を除外）
2. 打刻ログから本日分のレコードを抽出
3. 各スタッフの状態を判定

**戻り値:**
```javascript
[
  { name: "山田 太郎", status: "none",    clockIn: null,   clockOut: null },
  { name: "佐藤 花子", status: "working", clockIn: "09:02", clockOut: null },
  { name: "田中 一郎", status: "done",    clockIn: "08:55", clockOut: "18:15" }
]
```

- status: "none"（未出勤）/ "working"（勤務中）/ "done"（退勤済）

### 4.4 promptNewMonth の仕様

1. 年と月をダイアログで入力受付
2. 確認ダイアログ（備考欄がクリアされる旨を通知）
3. 全スタッフシートのD1（年）とF1（月）を更新
4. H4:H34（備考欄）をクリア
5. 打刻ログの生データはそのまま保持

### 4.5 設定値（定数）

```javascript
const SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID_HERE'; // ← スプレッドシートURLからIDを取得して設定（必須）
const STAFF_NAMES = ['山田 太郎', '佐藤 花子', '田中 一郎']; // 初期設定時のみ使用
const LOG_SHEET_NAME = '打刻ログ';
const OVERTIME_THRESHOLD = 8;       // 残業基準: 1日8時間超
const LUNCH_DEDUCT_6H = 0.75;      // 6時間以上勤務: 45分控除
const LUNCH_DEDUCT_8H = 1.0;       // 8時間以上勤務: 60分控除
```

## 5. フロントエンド仕様（Index.html）

### 5.1 概要

Google Apps Script の `doGet()` から返されるHTML。タブレット端末のブラウザでフルスクリーン表示を想定。GASの `google.script.run` を使ってサーバー関数を呼び出す。

**重要:** このHTMLファイルはGAS内の `HtmlService.createHtmlOutputFromFile('Index')` で読み込まれるため、ファイル名は必ず `Index`（拡張子なし、GASエディタ上では Index.html）にする。

### 5.2 画面構成

```
┌─────────────────────────────────────────────┐
│ [TC] タイムカード                  ● オンライン │  ← ヘッダー
├─────────────────────────────────────────────┤
│                                             │
│                  10:03                      │  ← 現在時刻（大きく表示）
│           2026年4月5日（日）                  │  ← 日付
│                                             │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │ 山田 太郎 │ │ 佐藤 花子 │ │ 田中 一郎 │    │  ← スタッフカード
│  │ ● 未出勤  │ │ ● 勤務中  │ │ ● 退勤済  │    │
│  │   入室→   │ │   ←退室  │ │          │    │
│  └──────────┘ └──────────┘ └──────────┘    │
│                                             │
└─────────────────────────────────────────────┘
```

### 5.3 UI動作フロー

1. **画面読み込み時:** `google.script.run.getStaffWithStatus()` でスタッフ一覧＋ステータスを取得
2. **カードタップ:** 未出勤 → 入室確認ダイアログ表示 / 勤務中 → 退室確認ダイアログ表示 / 退勤済 → 操作不可
3. **確認ダイアログ:**
   - 種別（入室/退室）、現在時刻、スタッフ名を表示
   - 「キャンセル」または「入室する/退室する」ボタン
4. **打刻実行:** `google.script.run.recordClock(name, type)` を呼び出し
5. **成功時:** トースト通知 → スタッフ一覧を再取得して画面更新
6. **失敗時:** エラーメッセージをトースト通知
7. **自動更新:** 5分ごとにスタッフ一覧を再取得

### 5.4 既知のバグと修正点

**バグ:** `confirmClock()` 関数内で `closeModal()` を呼んだ後に `pendingAction` を参照していた。`closeModal()` は `pendingAction = null` にリセットするため、その後の `pendingAction.name` でクラッシュ。

**修正:** `closeModal()` を呼ぶ前に `name` と `type` をローカル変数に退避する。

```javascript
// 修正済みコード
function confirmClock() {
    if (!pendingAction || isProcessing) return;
    var name = pendingAction.name;   // ← 先に退避
    var type = pendingAction.type;   // ← 先に退避
    isProcessing = true;
    closeModal();                    // ← ここで pendingAction が null になる
    renderStaff();
    google.script.run
      .withSuccessHandler(function(result) { ... })
      .withFailureHandler(function(err) { ... })
      .recordClock(name, type);
}
```

### 5.5 デザイン仕様

- **テーマ:** ダークモード（背景 #070e1a）
- **フォント:** system-ui, Hiragino Sans, Noto Sans JP
- **時計:** 72px、200 weight、タブラー数字
- **カードレイアウト:** CSS Grid、minmax(300px, 1fr)
- **ステータス表示:**
  - 未出勤: グレーバッジ
  - 勤務中: 緑バッジ（脈動アニメーション付き）、カード上部に緑のアクセントライン
  - 退勤済: 黄バッジ、カード半透明、操作不可
- **接続状態:** ヘッダー右に緑/赤ドットで表示
- **トースト通知:** 画面下部中央、成功＝緑、エラー＝赤、3秒後自動消去
- **確認ダイアログ:** 中央モーダル、背景ぼかし、時刻を大きく表示

## 6. 集計ルール

| ルール | 内容 |
|--------|------|
| 残業基準 | 1日8時間超を残業として計上 |
| 昼休み控除 | 6時間以上勤務 → 45分控除 / 8時間以上勤務 → 60分控除 |
| 有給・イレギュラー | 総務が備考欄に手入力 → 社労士提出時に反映 |
| 月の日数 | EOMONTH関数で自動判定（28日〜31日対応） |

## 7. 運用フロー

### 7.1 日常（スタッフ操作）
1. タブレット端末のブラウザで打刻画面を開く
2. 自分の名前カードをタップ → 「入室する」を押す
3. 退勤時に再度タップ → 「退室する」を押す

### 7.2 月末（総務操作）
1. Googleスプレッドシートをブラウザで開く
2. 各スタッフのシートを確認
3. 打刻漏れがあれば入室・退室セルを手入力で修正
4. 有給の日は備考欄に「有給」と記入
5. 下段の月次集計値を社労士提出フォーマットに転記
6. CSV または PDF で出力して社労士へ提出

### 7.3 月初（総務操作）
- メニュー「勤怠管理 → 新しい月へ移行」で年月を更新
- または各シートのD1/F1セルを直接変更
- 前月のデータが必要なら先にCSVダウンロードしておく

### 7.4 スタッフの追加・変更
- 既存スタッフシートのタブを右クリック →「コピーを作成」
- タブ名とA1セルを新スタッフ名に変更
- 備考欄をクリア

## 9. 技術メモ

### 9.1 GAS固有の注意点

- `google.script.run` はクライアントサイドからGAS関数を非同期で呼び出すAPI
- `doGet()` が返す `HtmlService.createHtmlOutputFromFile('Index')` はファイル名が完全一致する必要あり
- Webアプリのデプロイ後、コード変更を反映するには「新しいバージョン」として再デプロイが必要
- 無料のGoogleアカウントでも利用可能

### 9.2 スプレッドシートの地域設定

- 「ファイル → 設定 → 地域」を「日本」に設定すること
- TEXT関数の "aaa" は地域が日本でないと曜日が正しく表示されない

### 9.3 制限事項

- GASの実行時間制限: 1回あたり6分（通常の打刻なら問題なし）
- 同時アクセス: 複数スタッフが完全同時に打刻した場合、appendRowは排他制御されるが、重複チェックにわずかなレースコンディションの可能性あり（5名以下の運用では実質問題にならない）
